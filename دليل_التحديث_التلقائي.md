# الدليل النهائي: TechnoAfandi - FC — من الفولدر الجاهز لحد التحديث شغال

الريبو: `https://github.com/mabdelmoneam03/TecchnoAfandi-FC`

انت دلوقتي عامل فولدر جديد وناقل فيه ملفات المشروع. اتبع بالظبط من هنا.

---

## 1) اتأكد إن الملفات في مكانها

من جوه الفولدر الجديد، المفروض يبقى عندك:

```
package.json                        🟢 عندك أصلاً
package-lock.json                   🟢 عندك أصلاً
vite.config.js                      🟢 عندك أصلاً
public/                             🟢 عندك أصلاً
src/
├── home_page_v2.html                🔵 من عندي (فيه relaunch)
└── version_page_v2.html             🟢 عندك أصلاً
src-tauri/
├── Cargo.toml                       🔵 من عندي (فيه updater/process)
├── Cargo.lock                       🟢 عندك أصلاً
├── build.rs                         🟢 عندك أصلاً
├── tauri.conf.json                  🔵 من عندي (فيه updater config)
├── icons/                           🟢 عندك أصلاً
├── capabilities/default.json        🔵 من عندي (ملف جديد)
├── assets/activator.exe             🔵 من عندي (نفس ملفك)
└── src/
    ├── main.rs                       🔵 من عندي
    └── activation.rs                 🟢 عندك أصلاً
.github/workflows/release.yml        🔵 من عندي (أحدث نسخة — بايباس لباج tauri-action)
```

---

## 2) عدّل حاجتين في `tauri.conf.json`

```json
"endpoints": ["https://github.com/mabdelmoneam03/TecchnoAfandi-FC/releases/latest/download/latest.json"]
```
(متظبط أصلاً في الملف اللي عندك)

```json
"pubkey": "PASTE_YOUR_PUBLIC_KEY_HERE"
```
سيبها لحد خطوة 5.

**تأكد إن `productName` من غير مسافات خالص** (`TechnoAfandi-FC`) — ده شرط أساسي.

---

## 3) ارفع الملفات على GitHub

من جوه الفولدر (التيرمينال):

```bash
git init
git remote add origin https://github.com/mabdelmoneam03/TecchnoAfandi-FC.git
git branch -M main
git add .
git commit -m "Add auto-update system"
git push -u origin main
```

### لو ظهرت رسائل خطأ

| الرسالة | الحل |
|---|---|
| `fatal: not a git repository` | نسيت `git init` — نفّذه الأول |
| `remote origin already exists` | استخدم `git remote set-url origin <الرابط>` بدل `add` |
| `rejected` أو `unrelated histories` | نفّذ `git pull origin main --allow-unrelated-histories` وبعدها كرر الـ push |

---

## 4) تأكد بعينك على GitHub

افتح الريبو وشوف إن `src-tauri/assets/activator.exe` و `src-tauri/icons/` و `.github/workflows/release.yml` كلهم وصلوا.

---

## 5) ولّد مفتاح التوقيع (لو لسه معملتوش)

```bash
npx @tauri-apps/cli signer generate -w ~/.tauri/technoafandi.key
```

هيديك **Private Key** و **Public Key**. متشاركش الـ Private Key مع حد.

---

## 6) حط الـ Secrets + الـ Public Key

**Settings → Secrets and variables → Actions** في الريبو:

| Name | Value |
|---|---|
| `TAURI_SIGNING_PRIVATE_KEY` | محتوى الـ Private Key كامل |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | كلمة السر (فاضية لو معملتش واحدة) |

بعدين افتح `tauri.conf.json` على GitHub، استبدل `PASTE_YOUR_PUBLIC_KEY_HERE` بالـ Public Key الحقيقي، Commit.

---

## 7) أول تاج (Tag)

```bash
git tag v1.0.0
git push origin v1.0.0
```

**قاعدة مهمة**: لو فشل أي حاجة، استخدم رقم جديد كليًا (`v1.0.1`) في المرة الجاية بدل ما تعيد استخدام نفس الرقم — امسح أي Release/Tag فاشل من GitHub قبل أي محاولة جديدة.

روح لتبويب **Actions** على GitHub مباشرة وتابع من هناك.

---

## 8) لو فشل الـ Build

- **"No artifacts were found"** → استخدمنا `release.yml` بيبني بـ `npx tauri build` مباشرة وبيرفع بأداة تانية (`softprops/action-gh-release`) بدل ما يعتمد على آلية البحث الداخلية البايظة في `tauri-action`. لو ده لسه بيحصل، ابعتلي اللوج كامل.
- **"signing key" error** → تأكد من محتوى الـ Secret، من غير مسافات زيادة
- **فشل بسبب activator.exe أو الأيقونات** → تأكد إنهم موجودين فعليًا على GitHub

---

## 9) جرّب التحديث فعليًا

1. نزّل installer نسخة `v1.0.0` من Releases وشغّله (النسخة "القديمة")
2. زوّد رقم النسخة في `tauri.conf.json` و `Cargo.toml` لـ `1.0.1`
3. ```bash
   git add .
   git commit -m "v1.0.1"
   git push
   git tag v1.0.1
   git push origin v1.0.1
   ```
4. سيب الـ Action يخلص
5. افتح النسخة القديمة، دوس Continue → المفروض يطلع البوب أب 📢 → دوس UPDATE
